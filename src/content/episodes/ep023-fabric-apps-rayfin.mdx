## Key Takeaways

- ▸ SkillOpt is framed as a way to scientifically evaluate skills by comparing outcomes, latency, and token usage across models and coding harnesses.
- ▸ The hosts stress that “good” skills require explicit success criteria and repeatable benchmarks, not just anecdotal demos.
- ▸ Fable is discussed as potentially better on some tasks but notably more expensive, making model routing and cost-aware usage essential.
- ▸ Rayfin templates are not identical: the data app template includes extra skills and tooling (like Playwright) beyond the base Rayfin skill.
- ▸ For semantic-model-connected apps, local development can be limited; deploying to Fabric and iterating against the hosted app can be a practical workaround.
