## Summary

In the first episode of Agentic Thinking, Mike Carlo and Mathias Thierbach use the idea of a mind palace to build a shared mental model for understanding AI agents. They separate simple chat experiences from truly agentic systems by focusing on tools, side effects, and the loop between prompt and action. A big theme is that the surrounding harness matters as much as the model itself, whether that harness is a coding tool, a desktop agent, or a visual workflow builder. They also argue that agentic tooling is lowering the barrier for non-developers to build useful software and automation much faster than before.

## Key Takeaways

- ▸ An agent is more than a chatbot because it can use tools, take actions, and iterate toward an outcome.
- ▸ The harness around a model often determines how useful an agent feels in practice.
- ▸ The biggest wins come from using agents to create and automate work, not just answer questions.
- ▸ The current tooling landscape is changing quickly, so teams need an updated mental model to keep up.
- ▸ Visual and natural-language workflows are making software creation more accessible to non-developers.
- ▸ Tools like n8n point toward a more flexible way to build agentic business processes.
