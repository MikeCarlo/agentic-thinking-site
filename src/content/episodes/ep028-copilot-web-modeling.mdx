## Summary

Jacinda introduces Copilot in web modeling (preview) for Power BI Service, an in-product assistant built on the same modeling MCP server capabilities many users already know. The episode shows how it analyzes semantic models, recommends fixes, and applies changes such as missing relationships, hidden technical columns, and cleanup of duplicate fields. A key theme is better model discoverability: clearer naming, richer descriptions, and captured business context so both people and agents can use models more effectively. They also preview upcoming “prep data for AI” support to suggest and apply AI instructions/schema and invite community feedback to shape the feature.

## Key Takeaways

- ▸ Copilot in web modeling brings MCP-powered semantic model tooling directly into Power BI Service with no separate local install.
- ▸ It can detect and propose concrete structural fixes, including missing table relationships and model hygiene improvements.
- ▸ The workflow is permissioned and transparent, showing reasoning steps and requiring confirmation before applying edits.
- ▸ Beyond structure, it helps improve discoverability through naming, descriptions, display folders, and business-logic context.
- ▸ “Prep data for AI” (coming soon in this flow) is positioned as the place to manage AI instructions and schema guidance for model readiness.
- ▸ The team explicitly asks users to share feedback via docs/blog/community channels to guide the next iteration of capabilities.
