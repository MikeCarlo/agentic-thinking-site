## Summary

Mike and Mathias return to issues they created in earlier episodes and use GitHub Copilot-style agents to work through them live. Before the demo, they cover Microsoft Foundry news, including hosted agents and toolboxes, and mix in a quiz segment to test ideas and assumptions. The core of the episode focuses on how agents approach real GitHub issues, what happens when they generate plans before coding, and how those plans can become unexpectedly strange or overly confident. It is a practical look at agent-assisted issue resolution, with plenty of commentary on the current state of planning, execution, and developer oversight.

## Key Takeaways

- ▸ Agents are now capable of picking up previously created GitHub Issues and making meaningful progress on them with relatively little manual setup.
- ▸ Microsoft Foundry's hosted agents and toolbox announcements point toward a more structured ecosystem for running and equipping agents at scale.
- ▸ Quiz-style prompting is a useful way to surface assumptions and test how well people understand fast-moving agentic tooling.
- ▸ Asking an agent to plan before it implements can improve structure, but it can also trigger odd behaviors and unrealistic assumptions.
- ▸ Time estimates and step-by-step implementation plans generated by agents should be treated as guidance, not as trustworthy project management artifacts.
- ▸ Even when agents can resolve issues quickly, humans still need to steer, review, and validate the work they produce.
