## Summary

In this episode, he builds two Rayfin templates end-to-end from Microsoft Fabric into VS Code: a basic authenticated app (Template 01) and a semantic-model-connected data app (Template 02). He shows the full workflow of scaffolding with `npx create rayfin`, running locally with `npm run dev`, signing in through Microsoft auth, and deploying with `npx rayfin up`. Along the way, he compares template capabilities, including the larger skill set in Template 02 and how Rayfin behaves like a static web app with built-in Fabric integration. He also troubleshoots install/deployment friction and UI styling issues while still getting the semantic model visuals published in Fabric.

## Key Takeaways

- ▸ Rayfin app creation starts in Fabric, then the generated command is run locally to scaffold the project in VS Code.
- ▸ Template 01 gives a minimal authenticated React/Vite starter, while Template 02 introduces data-app patterns and many more built-in agent skills.
- ▸ A failed dependency step during scaffolding can be recovered by manually running `npm install` in the template folder.
- ▸ `npx rayfin up` refreshes environment config, builds static assets, and republishes changes directly to the Fabric item.
- ▸ The host distinguishes app management from code publishing: settings are in Fabric, but real development and CI/CD stay in source control/VS Code.
- ▸ Agent-driven UI edits can accelerate iteration, but model/tooling mismatches (like dark/light mode behavior) still require debugging and validation.
