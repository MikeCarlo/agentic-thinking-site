## Summary

Mike and Mathias pick up where last week left off — using the Microsoft Fabric MCP modeling server as an NPM package inside GitHub Copilot — and push it much further with a live demo. Mathias connects Copilot to a remote semantic model in Fabric, asks the agent to review and document it, export a TMSL dump, and then automatically create GitHub Issues for every identified improvement of medium severity or higher. Along the way they discuss why markdown-first documentation and front-loaded planning are dramatically improving their agentic workflows. The episode also covers the just-released Codex 5.5 and what early users are seeing from it.

## Key Takeaways

- ▸ An agent connected to the Fabric MCP modeling server can review an entire Power BI semantic model and produce a prioritized improvement list in a single premium request.
- ▸ Exporting documentation as markdown rather than Word or Excel significantly reduces token usage when feeding context to agents.
- ▸ Front-loading the planning phase — writing clear phase-by-phase specs before handing work to an agent — consistently improves output quality.
- ▸ Codex 5.5 is available in the Codex CLI but ignoring architectural constraints in specs was an early disappointment for Mathias in initial testing.
- ▸ GitLens is a must-have VS Code extension for anyone working with Git, offering a richer experience than the built-in source control panel.
- ▸ Rui Romano's Power BI agentic plugins are the next frontier to explore for Fabric + Copilot workflows.
