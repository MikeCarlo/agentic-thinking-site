## Summary

Mike and Mathias open with a short discussion of "the age of AI - Founder Mode" before shifting into a practical walkthrough on preparing Power BI semantic models for agentic workflows. The core argument is that agents need more than syntactically correct models — they need metadata that explains business meaning, not just technical structure. The episode focuses on agent-friendly naming, richer table and measure descriptions, and annotations that help an agent understand what a DAX measure is for in the context of a report. They round it out with a live demo showing how small metadata improvements make a sample model easier for agents to navigate and use.

## Key Takeaways

- ▸ AI agents perform better against Power BI models when names are explicit, consistent, and descriptive.
- ▸ Semantic model descriptions should explain business purpose and reporting context, not just technical definitions.
- ▸ DAX measures need annotations that tell an agent what the metric means and when it should be used.
- ▸ Poor metadata forces agents to infer intent from formulas alone, which leads to weaker analysis.
- ▸ Optimizing a model for agents is largely a metadata quality exercise rather than a modeling overhaul.
- ▸ A live cleanup demo shows how incremental documentation improvements can materially improve agent navigation.
