## Summary

Michael and Matthias run a wide-ranging news episode covering OpenAI’s strategy update, Build announcements, and what they see as the shift from model novelty to practical optimization. They connect that trend to real engineering tradeoffs: local inference for privacy and latency, token economics, and choosing the right model/harness/reasoning level for each task. The discussion also revisits Rayfin as a Fabric backend pattern, including current limitations (for example, custom domains and backend extensibility) and emerging opportunities for ISV templates and enterprise reuse. They close by stressing that teams need to capture session telemetry and outcomes if they want to systematically improve agent performance and cost.

## Key Takeaways

- ▸ The hosts describe the current moment as an optimization phase: the hard problem is no longer access to models, but routing tasks to the most efficient model stack.
- ▸ For analytics workflows, they stress keeping semantic models and skill docs in sync, because stale skill context can cause accuracy drift.
- ▸ They argue metric definitions should stay human-owned, with agents assisting on documentation and implementation rather than inventing business logic.
- ▸ Rayfin is treated as foundational Fabric backend infrastructure for app experiences, but still has preview-stage constraints around domains and backend patterns.
- ▸ Sustainable agent operations require data: collect session history, correlate with commits/tasks, and use experiments to tune cost versus quality over time.
