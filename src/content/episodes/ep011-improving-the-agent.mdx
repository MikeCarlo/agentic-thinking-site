## Summary

Mathias and Mike continue their hands-on series on improving agent output, starting with news about Anthropic’s pricing changes and the practical cost implications of agentic engineering. From there, they compare how they’re using different harnesses — especially VS Code, CLI workflows, and headless automation — before revisiting their Power BI semantic model experiment from the prior episode. The core demo shows how repo-level instructions and a purpose-built local skill can push an agent toward better semantic descriptions, cleaner issue-oriented work, and more reusable behavior. They close by arguing that any repeatable prompt should eventually become a skill or custom agent, and that these artifacts should evolve alongside the project.

## Key Takeaways

- ▸ Agentic workflows are increasingly shaped by real cost and capacity constraints, not just model capability.
- ▸ Different harnesses excel at different jobs: interactive chat is useful, but headless and automated flows unlock more scalable engineering patterns.
- ▸ `agents.md` can improve baseline behavior, but broad repo instructions also add context to every agent interaction.
- ▸ A narrowly scoped local skill significantly improved the semantic-model documentation task compared with reusing the same raw prompt alone.
- ▸ Skill descriptions matter twice: they help humans understand intent and determine whether the model will auto-invoke the skill.
- ▸ If a workflow is repeatable, it should probably become a reusable skill, custom agent, or other project artifact instead of staying a one-off prompt.
