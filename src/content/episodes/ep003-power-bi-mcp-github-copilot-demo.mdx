## Summary

Mike Carlo and Mathias Thierbach open with Anthropic and GitHub news before shifting into a live demo of GitHub Copilot cloud agents working with the Power BI Modeling MCP server. They spend time on a key theme from the episode: the harness and routing strategy around a model often matters more than the model itself. The demo walks through the newer NPM-based setup, how the agent connects to a Power BI model, and how MCP tools show up inside the GitHub experience. They also unpack premium request pricing, secret-passing rules, and the practical differences between local MCP servers and easier HTTP-based setups.

## Key Takeaways

- ▸ Harness design and model routing can matter more than simply choosing the strongest model.
- ▸ GitHub Copilot cloud agents can run Power BI MCP-backed tasks remotely against a repo-based workflow.
- ▸ The Power BI Modeling MCP server being available as an NPM package makes non-VS Code setup much easier.
- ▸ Copilot premium requests are easier to reason about than opaque token-based usage windows.
- ▸ Secrets must be explicitly passed through the Copilot environment, and MCP-related secrets need the `copilot_mcp_` prefix.
- ▸ MCP tool access can be tightened for safety, which matters when exposing modeling actions to agents.
