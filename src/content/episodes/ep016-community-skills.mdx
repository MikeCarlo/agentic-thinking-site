## Summary

This episode is a hands-on walkthrough of community skills: where to find them, how to install them in VS Code, and how to use them safely in real workflows. Mike and Matthias compare large marketplaces, then show the actual plugin plumbing (marketplace.json, plugin folders, and user-level marketplace settings). They demo adding Microsoft Fabric skills from source and testing skill invocation behavior in chat, including explicit slash-command targeting. A key theme is governance: too many overlapping skills can bloat context and create inconsistent results across machines. The practical advice is to install selectively, inspect every skill.md, and treat skills as assets you actively curate.

## Key Takeaways

- ▸ Skills are effectively packaged prompt instructions, and modern agent workflows increasingly depend on discovering and curating the right ones.
- ▸ VS Code plugin discovery can be non-obvious (for example, using `@agent plugins`), so knowing the search and install paths is part of effective setup.
- ▸ Marketplaces are configurable at the user level, and teams can publish their own shared plugin repositories to standardize skills across developers.
- ▸ Installing from source (owner/repo) makes it easy to add external skill packs such as Microsoft’s Fabric skills and inspect exactly what was pulled locally.
- ▸ For model selection, the skill description is critical: the agent first sees that description and decides whether to invoke the skill, unless you force usage with slash commands.
- ▸ Avoid skill sprawl—review skill.md files, keep descriptions tight, and add plugins gradually to reduce conflicts, hidden context bloat, and inconsistent agent behavior.
