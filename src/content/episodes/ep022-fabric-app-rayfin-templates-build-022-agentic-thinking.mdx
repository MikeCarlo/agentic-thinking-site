## Summary

In this longer build session, he creates Rayfin Template 03 (the to-do app) in Fabric, runs it locally, and publishes it to the service. He then inspects the generated SQL database artifacts and uses Copilot Ask mode to trace how the app talks to data through Rayfin’s GraphQL/Data API Builder layer rather than direct browser-to-SQL access. The episode also demonstrates adding the Agentation workflow so UI feedback can be placed directly on the running app and converted into agent tasks. Several UI refinements are applied, but dark/light mode behavior remains partially broken by the end, illustrating real-world agentic debugging limits.

## Key Takeaways

- ▸ Template 03 scaffolds a production-oriented to-do app with Fabric auth, local dev workflow, and deploy-ready structure.
- ▸ Publishing with `npx rayfin up` creates and updates both the app and related backend artifacts visible in the Fabric workspace.
- ▸ The generated data path is app → Rayfin API/GraphQL layer → SQL, not direct client-side SQL connectivity.
- ▸ Table/schema definitions can be traced in project entity files, which map to the SQL objects created for the app.
- ▸ Agentation-style visual annotation can speed UI iteration by turning on-page comments into actionable agent tasks.
- ▸ Even with strong agent support, CSS/theme issues may need multiple model passes and can still ship with known limitations.
