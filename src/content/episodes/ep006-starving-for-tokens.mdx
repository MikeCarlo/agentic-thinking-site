## Summary

GitHub's announcement of usage-based Copilot pricing is the headline — and both hosts predicted it. The flat-rate unlimited AI era is ending, and developers now face real per-token costs passed directly from model providers. Mike and Mathias walk through what the pricing shift actually means in dollars, compare it to direct API pricing for models like GPT-4.5 and Opus 4.7, and discuss how engineering teams need to adapt. The conversation moves into practical optimization: routing tasks to cheaper models, understanding input versus output token costs, building observability into every agent workflow, and the emergence of the "agent engineer" as a new technical role.

## Key Takeaways

- ▸ GitHub Copilot is moving from premium request bundles to full retail API pricing pass-through, making the cost of using top-tier models immediately visible to subscribers.
- ▸ Not every agentic task needs a frontier model — routing simple subtasks to Haiku, mini, or nano models can dramatically reduce costs without sacrificing quality.
- ▸ Input tokens (context sent to the model) are typically the bigger cost driver; keeping repository context lean and well-structured pays off.
- ▸ Observability is now a first-class engineering requirement: every agent session trace you fail to record is a lost opportunity to optimize spend.
- ▸ Companies that relied on in-house software engineering teams will likely evolve toward agent engineering roles focused on orchestration, routing, and cost governance.
