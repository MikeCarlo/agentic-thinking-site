## Summary

This episode focuses on using Rayfin Template 03 with an external Git template source from the Awesome Rayfin repository. He first hits a template fetch error by pointing too deep into the repo, then fixes it by using the repository root URL so the CLI can list available templates. After selecting the IBCS Trainer template, he scaffolds the project locally, installs dependencies, and deploys with `npx rayfin up`. The walkthrough ends with a live Fabric run of the deployed app, including gameplay-style interactions that confirm the template is working.

## Key Takeaways

- ▸ For external templates, point Rayfin CLI to the root Git repo, not a nested folder URL, to avoid fetch failures.
- ▸ Template 03 can scaffold from community templates and then sync with current Rayfin scaffolding automatically.
- ▸ The Awesome Rayfin gallery provides multiple React/Tailwind-based starter apps that can be selected during CLI setup.
- ▸ `npx rayfin up` packages the local build, creates the distribution artifacts, and pushes the app to Fabric.
- ▸ Deployment status can be verified by watching build/deploy logs and then refreshing the Fabric app item.
- ▸ A successful publish should result in an immediately usable in-service app experience, not just a local dev preview.
